# mykindle

Утилита для подготовки книг и документов и заливки их на Kindle — **локально,
без интернета на самом устройстве** (не через Amazon Send-to-Kindle).

Ты складываешь файлы в папки, утилита конвертирует их в Kindle-формат, проставляет
теги (чтобы поиск на самом Kindle их находил) и заливает на устройство по USB.

**Платформы:** сейчас — **macOS** (Calibre + `mtp-cli`). Архитектура переносимая,
Linux/Windows — в планах (конвертация через Calibre кроссплатформенна, заливка —
через `libmtp`).

## Зачем

- Kindle часто без интернета → заливаем по кабелю, а не облаком.
- Нужны не только книги, но и документы (PDF, DOCX, …).
- На новых Kindle коллекции заводятся только тапами на e-ink. Вместо них — **теги
  в метаданных**: поиск на Kindle по тегу находит нужное, сортировка «По автору»
  группирует.

## Установка / обновление

```bash
git clone https://github.com/kscht/mykindle.git ~/mykindle
cd ~/mykindle
# обновиться позже:
git pull
```

Требуется на машине:
- **Python 3** (сам инструмент `bin/kindle` — на Python, без сторонних зависимостей);
- [Calibre](https://calibre-ebook.com/) (CLI: `ebook-convert`, `ebook-meta`, `calibredb`);
- для заливки на macOS — `mtp-cli` (внутри [OpenMTP](https://github.com/ganeshrvel/openmtp));
  на Linux — `libmtp` (`mtp-sendfile`);
- для богатого форматирования (KFX) — Kindle Previewer + KFX-плагины Calibre.

📦 **Установка на чистый Mac пошагово — [`docs/setup-macos.md`](docs/setup-macos.md).**

## Как пользоваться

1. **Раскладывай файлы по папкам** в `inbox/`. Имена папок — любые, как удобно
   тебе (по предметам, проектам, темам). На обработку имена папок не влияют.
2. **Теги задаёшь служебным файлом `_kindle.md`** в нужной папке (см. ниже).
3. **Конвертируешь и заливаешь командой `kindle`** (см. ниже). Заливка требует
   физически подключённого Kindle.

### Команды

Запускаются из `~/mykindle`:

```bash
./bin/kindle status            # что в inbox / staging / на устройстве
./bin/kindle convert           # inbox → staging: конвертация + впекание тегов
./bin/kindle convert --jobs 6  # то же в 6 параллельных потоков (заметно быстрее)
./bin/kindle convert Пушкин    # только файлы, в пути которых есть «Пушкин»
./bin/kindle push              # staging → Kindle /documents (нужен воткнутый Kindle)
./bin/kindle sync              # convert + push разом
```

**Форматы вывода подбираются автоматически:**
- **KFX** — по умолчанию: богатое форматирование (книги, таблицы, стили);
- **MOBI6** — авто-фолбэк, если KFX не собрался; максимальная совместимость;
- **PDF** — заливается как есть (для сложной вёрстки/сканов; теги — в имя файла).

> Почему так: на новых прошивках Kindle сайдлоад **AZW3 даёт чёрный экран**, а
> EPUB не распознаётся. Рабочие форматы — **KFX** (богатое) и **MOBI6** (простое).

По умолчанию **встроенные в книгу шрифты выкидываются** — в книгах часто лежат
неполные subset-шрифты, из-за которых на Kindle появляются «квадратики» (◻) вместо
букв; без них устройство берёт свои полные шрифты (и ты можешь менять шрифт сам).

Тонкая настройка через env: `KINDLE_JOBS` (число потоков), `KINDLE_RICH_FORMAT=mobi`
(гнать MOBI вместо KFX), `KINDLE_KEEP_FONTS=1` (оставить встроенные шрифты),
`KINDLE_PROFILE`, `CALIBRE_BIN`, `MTP_CLI`.

### Файл настроек папки `_kindle.md`

В любой папке `inbox/` можно положить `_kindle.md` — он задаёт обработку для всех
файлов этой папки (и вложенных). YAML-шапка + свободные заметки:

```markdown
---
add_authors: [чтение, лето, 7кл]   # доп-«авторы» = теги (для поиска на Kindle)
type: книга                         # книга | документ
---
Свободный текст — твои заметки, на обработку не влияют.
```

**Каскад по вложенности:** теги из `_kindle.md` всех родительских папок
**накапливаются** вниз (объединяются, дубли убираются); `type` берётся из
ближайшей папки. Пример:

```
inbox/Школа/            _kindle.md → add_authors: [Школа, 7кл]
inbox/Школа/Лето/       _kindle.md → add_authors: [лето, чтение]
inbox/Школа/Лето/война-и-мир.epub
   → на Kindle «авторы»: Толстой & Школа & 7кл & лето & чтение
```

Файл без единого `_kindle.md` по пути → заливается как есть, без тегов.

### Подробный пример: как разложить книги (можно копировать)

Шаг 1. Создай в `inbox/` такие папки (имена — любые, как тебе удобно):

```
inbox/
└── Школа/
    ├── _kindle.md
    ├── Лето/
    │   ├── _kindle.md
    │   └── (сюда кидаешь книги летнего чтения)
    └── 7 класс/
        ├── _kindle.md
        └── (сюда кидаешь учебники 7 класса)
```

Шаг 2. В каждую папку положи файл `_kindle.md` с таким содержимым (скопируй ровно):

Файл `inbox/Школа/_kindle.md`:
```markdown
---
add_authors: [Школа]
type: книга
---
Всё школьное.
```

Файл `inbox/Школа/Лето/_kindle.md`:
```markdown
---
add_authors: [Каникулы]
type: книга
---
Летнее чтение.
```

Файл `inbox/Школа/7 класс/_kindle.md`:
```markdown
---
add_authors: [седьмойкл]
type: книга
---
7 класс.
```

Шаг 3. Что получится (теги родительских папок **добавляются** к вложенным):

| Куда положил книгу | Теги-«авторы» на Kindle |
|---|---|
| `inbox/Школа/` | `Школа` |
| `inbox/Школа/Лето/` | `Школа & Каникулы` |
| `inbox/Школа/7 класс/` | `Школа & седьмойкл` |

На Kindle поиск по `Каникулы` найдёт всё летнее чтение, по `седьмойкл` — всё за
7 класс, по `Школа` — вообще всё школьное.

> Подсказка: ты можешь просто попросить своего Claude в `~/mykindle` —
> «создай структуру папок Школа/Лето/7 класс с такими `_kindle.md`» — и он
> сделает файлы за тебя.

### Как теги попадают на Kindle

- Теги впекаются в **поле «Автор»** Kindle-копии (KFX/MOBI) при конвертации.
  Kindle индексирует авторов для поиска → ищешь по тегу, находишь файл.
- **Порядок в `add_authors` важен**: первым ставь главную «полку» — по первому
  автору идёт сортировка «По автору» на Kindle; поиск находит по любому.
- `книга` → `настоящий_автор & теги`; `документ` → `теги` (у документов автора
  обычно нет).
- Библиотека Calibre при этом остаётся с **чистыми** метаданными — теги идут
  только в копию, что уезжает на устройство.

## Структура

```
inbox/      сюда кладёшь файлы (по своим папкам, + _kindle.md)
staging/    сюда падают сконвертированные .kfx/.mobi/.pdf перед заливкой (служебное)
.state/     что уже залито на Kindle, для синхронизации (служебное)
bin/kindle  сам инструмент (Python 3)
```

`inbox/ staging/ .state/` содержат твой контент — он **в `.gitignore`**, `git pull`
его не трогает и не удаляет.
