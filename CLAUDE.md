# mykindle — заметки для разработки

Утилита: подготовка книг/документов и заливка на Kindle локально по USB
(без интернета на устройстве). Пользовательская документация — `README.md`.

## Цель и принципы

- **Оффлайн-first.** Конвертация и организация работают без сети; интернет нужен
  только чтобы обновить саму утилиту (`git pull`).
- **Переносимость.** Сейчас реализация под macOS, но OS-специфика (пути к Calibre,
  способ заливки по MTP) должна быть **за абстракцией/конфигом**, не захардкожена,
  чтобы добавить Linux/Windows. Не вшивай абсолютные macOS-пути в логику.
- **Чистая библиотека.** Метаданные в Calibre не пачкаем — теги впекаются только
  в Kindle-копию (KFX/MOBI), что уезжает на устройство.

## Pipeline

```
inbox/ (файлы по папкам + _kindle.md)
  → маршрутизация по типу: PDF — как есть; остальное → KFX (фолбэк MOBI6)
  → конвертация + впекание тегов в авторов (ebook-convert --authors)
  → staging/                                       [параллельно, --jobs N]
  → заливка на Kindle (mtp-cli put → /documents)   [нужен воткнутый Kindle]
  → .state/ (что уже залито — для sync)
```

Реализовано в **`bin/kindle`** (Python 3): команды `convert` / `push` / `status` / `sync`.

### Выходные форматы (проверено на устройстве)

Сайдлоад через `/documents` тестировался на живом Kindle:
- **KFX** — ✅ читается, богатое форматирование. **Дефолт для книг** (см. ниже).
- **MOBI6** — ✅ читается; **запасной** формат (авто-фолбэк, если KFX не собрался).
- **AZW3** — ❌ чёрный/пустой экран при открытии (баг прошивки на сайдлоаде; ASIN
  ни при чём — MOBI тоже с ASIN, но открывается).
- **EPUB** — ❌ не распознаётся при сайдлоаде в `/documents`.

`ebook-convert <in> out.mobi --output-profile kindle_pw`

⚠️ **Критично: только MOBI6 (`--mobi-file-type old`, это и есть дефолт Calibre —
«compatible with all devices»). НЕ использовать `both`/`new`** — они встраивают
**KF8** (= то же, что AZW3), а KF8 на этой прошивке битый (чёрный экран). AZW3
ломается системно на любых книгах (проверено пользователем), не из-за контента и
не из-за ASIN (MOBI тоже с ASIN, но работает). Размерный тест: `both` вдвое крупнее
`old` из-за KF8-секции.

Плата за MOBI6 — ограниченное богатое форматирование (простой CSS; таблицы
рендерятся, но без сложной вёрстки; без встраивания шрифтов).

### KFX — богатое форматирование (работает по CLI)

Поднят путь KFX: `ebook-convert in out.kfx --output-profile kindle_pw` через плагин
**KFX Output** + **Kindle Previewer 3** (ставятся, см. `docs/setup-macos.md`).
Проверено: конвертация идёт **headless по CLI** (Previewer сам в CLI-режиме, EULA
не спрашивает), теги-авторы сохраняются, таблицы — настоящие (`yj_table`).
Пайплайн: Ebook → Calibre(EPUB) → Previewer → KPF → KFX.

✅ **Рендер KFX проверен на устройстве** — открывается, богатое форматирование и
таблицы на месте (в отличие от чёрного экрана AZW3).

⚠️ **Шрифты (решено):** в книгах часто лежат **неполные subset-шрифты** (напр. в
fb2.epub был «Times New Roman» с жирным всего 23/64 кириллицы, латиница 1–5/26) →
на Kindle отсутствующие глифы рисуются «квадратиками» (tofu). Лечится **выкидыванием
встроенных шрифтов** при конвертации (`--filter-css "font-family,font"`, включено по
умолчанию; `KINDLE_KEEP_FONTS=1` отключает) → устройство берёт свои полные шрифты.
Подтверждено на боевой пачке. Для PDF со сложной вёрсткой по-прежнему лучше PDF.

**Итоговый выбор формата:** KFX — дефолт для богатого контента (книги, форматирование,
таблицы); MOBI6 — запасной/простой текст и максимальная совместимость; PDF — сложная
вёрстка/сканы (оставлять как есть, теги в имя файла).

## Формат `_kindle.md` (для реализации парсера)

- Зарезервированное имя `_kindle.md` в любой папке `inbox/`. Трактуется как
  **настройки**, не как контент (никогда не конвертируется/заливается).
- YAML-frontmatter (между `---`) + произвольный body (игнорируется обработкой).
- Поля v1:
  - `add_authors: [..]` — список тегов → доп-авторы.
  - `type: книга | документ` — книга сохраняет реального автора, документ нет.
- **Каскад:** для файла собираем `_kindle.md` по всему пути от `inbox/` до его
  папки. `add_authors` — объединение (сохраняя порядок: родитель → потомок),
  дедуп. `type` — из ближайшей папки (потомок переопределяет родителя).
- Нет ни одного `_kindle.md` по пути → без тегов, как есть.

## Сборка метаданных Kindle-копии

- `книга`: authors = `[реальный_автор] + add_authors` (реальный — из метаданных
  файла; если нет — `Unknown`).
- `документ`: authors = `add_authors` (если пусто — без изменений).
- Порядок важен: первый автор = группа при сортировке «По автору»; поиск находит
  по любому. Впекаем через `ebook-convert --authors "A & B & C"` в KFX/MOBI в `staging/`.
- (Опционально, позже) дублировать теги в title.

## Структура репо

```
inbox/ staging/ .state/   рабочие папки (контент в .gitignore, кроме .gitkeep)
bin/kindle                CLI-инструмент (Python 3): convert/push/status/sync
docs/setup-macos.md       установка на чистый Mac
README.md                 пользовательская дока
CLAUDE.md                 этот файл
```

## Безопасность репо

Репо **публичный** → никаких секретов, хостов, IP, учёток, личных данных.
Деплой/доступ к конкретной машине держатся **вне репо** (приватно). Локальный
приватный контекст (если есть) — в `CLAUDE.local.md` (в `.gitignore`).
